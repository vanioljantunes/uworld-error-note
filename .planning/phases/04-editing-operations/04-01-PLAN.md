  <behavior>
    - EDIT_NODE({id: "n0", label: "New Text"}) on a 3-node graph: node n0.label becomes "New Text", hasUserEdited is true
    - ADD_NODE({label: "New Box", afterId: "n1"}) on A->B->C chain: inserts new node between B and C, new edge from B to new node, new edge from new node to C, nodeCounter increments
    - ADD_NODE({label: "Tail"}) without afterId on A->B chain: appends to end, edge from B to new node
    - REMOVE_NODE({id: "n1"}) on A->B->C chain: removes B, reconnects A->C, edges cleaned
    - REMOVE_NODE on branch parent: removes node, cleans branchGroups referencing that parentId
    - ADD_EDGE({fromId: "n0", toId: "n2", stepLabel: "activates"}): creates edge, no duplicate if already exists
    - REMOVE_EDGE({fromId: "n0", toId: "n1"}): removes that specific edge
    - REORDER_NODE({id: "n1", direction: "up"}) on A->B->C: swaps labels of n0 and n1 (A becomes B's label, B becomes A's label)
    - REORDER_NODE at boundary (first node up, last node down): no-op
    - After any mutation action, hasUserEdited is true
    - LOAD action sets hasUserEdited to false
    - Node ID generation uses monotonic nodeCounter (no collisions after remove+add)
  </behavior>
