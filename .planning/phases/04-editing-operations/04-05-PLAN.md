1. Update the EdgePill function signature to accept `fromId`, `toId`, and `onRemove` props in addition to `label`:

```tsx
function EdgePill({
  label,
  fromId,
  toId,
  onRemove,
}: {
  label: string;
  fromId: string;
  toId: string;
  onRemove: (fromId: string, toId: string) => void;
}) {
  if (!label) return null;
  return (
    <div className={styles.pillWrap}>
      <div className={styles.pill}>{label}</div>
      <button
        className={styles.pillRemoveBtn}
        onClick={(e) => {
          e.stopPropagation();
          onRemove(fromId, toId);
        }}
        title="Remove connection"
        aria-label="Remove connection"
      >
        x
      </button>
    </div>
  );
}
```

Key design: The remove button (x) appears on hover of the `.pillWrap` container — same pattern as the node hover controls overlay (lines 291-320). `e.stopPropagation()` prevents event bubbling to connect-mode handlers.

For edges with empty labels (EdgePill returns null), the connection still cannot be removed via pill — this is acceptable for v1 since empty-label edges are structural connectors. If needed, a future iteration could show a minimal clickable dot for empty-label edges.
