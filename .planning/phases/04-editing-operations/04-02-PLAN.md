  <action>
    **1. Replace NodeCard with EditableNodeCard in FlowchartEditor.tsx:**

    Create `EditableNodeCard` component (per Research Pattern 1):
    - Props: `node: FlowNode`, `isEditing: boolean`, `isSelected: boolean`, `connectMode: boolean`, `onStartEdit: () => void`, `onCommit: (label: string) => void`, `onConnectClick: (nodeId: string) => void`, `onRemove: () => void`, `onReorder: (dir: "up" | "down") => void`, `isFirst: boolean`, `isLast: boolean`
    - When `isEditing`: render a `<textarea>` with className `styles.nodeCardTextarea`, autoFocus, value from local `draft` state (initialized from `node.label`), onBlur calls `onCommit(draft)`, Enter (without Shift) calls `onCommit(draft)`, spellCheck=false
    - When NOT editing: render the existing `<div className={styles.nodeCard}>` with `highlightCloze(node.label)`
    - onClick logic: if `connectMode` is active, call `onConnectClick(node.id)`. Otherwise call `onStartEdit()`.
    - Apply `styles.nodeCardSelected` className when `isSelected` (source node in connect mode)
    - On hover, show node controls overlay (`styles.nodeControls`):
      - Up arrow button (hidden if `isFirst`), dispatches `onReorder("up")`
      - Down arrow button (hidden if `isLast`), dispatches `onReorder("down")`
      - Delete button (x), dispatches `onRemove`
    - Use `cursor: "text"` when not in connect mode, `cursor: "crosshair"` when in connect mode

    **2. Add toolbar below header in FlowchartEditor:**

    Render a `<div className={styles.toolbar}>` between the header and the canvas (only when `viewMode === "editor"`):
    - "Add Box" button (`styles.toolbarBtn`): dispatches `ADD_NODE` with label `"New box"` and `afterId` set to `state.selectedNodeId` (or undefined if none selected)
    - "Connect" toggle button (`styles.toolbarBtn` + `styles.toolbarBtnActive` when `connectMode`): toggles `connectMode` in state. When toggling off, also clear `connectingFromId`.
    - "Delete" button (`styles.toolbarBtn`): dispatches `REMOVE_NODE` for `state.selectedNodeId` if one is selected; disabled if no selection

    **3. Wire connect mode in FlowRenderer:**

    In `FlowRenderer`, pass the connect mode state down to each `EditableNodeCard`. When `onConnectClick` fires:
    - If `connectingFromId` is null: set `connectingFromId` to the clicked node's id, and set `selectedNodeId` to that id
    - If `connectingFromId` is already set and the clicked node is different: prompt for step label via `window.prompt("Step label (optional):")`, then dispatch `ADD_EDGE`, clear `connectMode`, clear `connectingFromId`, clear `selectedNodeId`
    - If clicked node is the same as `connectingFromId`: cancel (clear `connectingFromId` and `selectedNodeId`)

    Note: Use `window.prompt` for the step label input. This is simple and adequate for v1. A custom popover is v2 polish.

    **4. Wire node selection for toolbar actions:**

    Clicking a node (when not in connect mode and not editing) should set `selectedNodeId` via a new action or by dispatching `SET_EDITING_NODE` followed by setting selectedNodeId. Actually, use a simpler approach: clicking a node in non-edit, non-connect mode starts editing. The "selected" state is only used for connect mode source highlighting. The toolbar "Add Box" button adds after the last node if no connect-mode selection.

    Simplification: "Add Box" always appends at the end of the chain (no afterId). "Delete" shows on each node's hover controls. This avoids needing a global selection state for non-connect purposes.
