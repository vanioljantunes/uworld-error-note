   - `FIXTURE_LINEAR`: A compact (no newlines between tags) HTML string of a linear 4-box flowchart. Build this by hand following the anki_flowchart template structure: outer wrapper div (text-align:center) > title div > box > stem > pill > stem > box > stem > pill > stem > box > stem > pill > stem > box. Include `{{c1::Thiamine deficiency}}` in the second box label and `{{c2::mammillary bodies::anatomy}}` in a later box label.
   - `FIXTURE_BRANCHING`: A compact HTML string with 1 root box, then a branch (display:inline-flex) into 2 children, each with a step pill and a box. Use left/right corner borders per the template.

   Write tests for the 5 behaviors listed above. Run tests — they MUST FAIL (RED phase).

2. THEN create `gapstrike/src/lib/parse-flow-html.ts`:
   - `export function parseFlowHTML(html: string): FlowGraph`
   - Use `new DOMParser().parseFromString(html, 'text/html')` to get a DOM
   - Get the wrapper: `doc.body.firstElementChild`
   - Implement a `getElementRole(el)` helper that identifies element type by style attribute matching against FLOWCHART_STYLES:
     - Check for title: style includes `font-weight:bold` AND `font-size:14px`
     - Check for box: style includes `display:inline-block` AND `border:2px solid #3a3a3a`
     - Check for branch: style includes `display:inline-flex`
     - Check for pill: style includes `display:inline-block` AND `font-size:10px`
     - Check for stemWrap: single child with style including `width:2px` AND `background:#3a3a3a`
     - Else: `unknown`
   - Walk wrapper's direct children sequentially. For each:
     - `title`: set `graph.title = el.textContent.trim()`
     - `box`: create FlowNode with `id: 'n' + nodeCounter++`, `label: el.textContent.trim()`. Push to nodes. If there's a pending pill label AND a previous node, create an edge from previous node to this node with that pill label.
     - `pill`: store label as pending (`el.textContent.trim()`)
     - `stemWrap`: skip (structural only)
     - `branch`: recursively process each child of the inline-flex div. Each child div contains corner + padding wrapper + (pill > stem > box)+ sequence. Create a BranchGroup with parentId = last non-branched node, childIds = nodes found in each branch child. Track the "last node" in the main chain so subsequent linear boxes connect correctly.
   - For box labels, use `el.textContent` (NOT `innerHTML`) — this preserves cloze syntax like `{{c1::text}}` which is plain text content, not HTML tags.
   - Return `{ title, nodes, edges, branchGroups }`.

3. Run tests — they MUST PASS (GREEN phase).

IMPORTANT: Do NOT normalize or lowercase style attribute values during matching. Match with `style.includes(...)` on the raw style string (collapse whitespace only). The AI template always emits exact style strings.

IMPORTANT: For branch parsing, the first child in the inline-flex wrapper is the LEFT child, last is the RIGHT child. Store childIds in this left-to-right order.
